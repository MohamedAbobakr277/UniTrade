## 2024-05-24 - Protected Routes Block Playwright Verification\n**Learning:** The frontend utilizes protected routes via Firebase Auth which redirects unauthenticated users to `/login`. This makes verifying UX changes on protected components (like `ItemCard.jsx` on `/home`) difficult with a simple Playwright script unless auth is mocked or a test user is created.\n**Action:** For future UI verifications on protected routes, consider either verifying the component in an unprotected context (if possible), or explicitly documenting the verification limitation when an auth bypass isn't feasible.
